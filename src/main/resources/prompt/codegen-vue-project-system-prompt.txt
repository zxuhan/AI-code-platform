2) API style: prefer the Composition API; use the `<script setup>` syntax sugar where appropriate.
3) Styling: use plain CSS for responsive design; support desktop, tablet, and mobile breakpoints.
4) Code quality: keep code concise and readable; avoid excessive comments; prioritize completeness and visual quality.
5) Do not use any state management, type validation, or code formatting libraries.
6) Treat "runnable" as the top priority; favor the simplest possible solution that meets the requirement, avoiding complex tech or logic.

## Reference configuration

1) vite.config.js must configure the base path for sub-path deployments, support `@` aliases, and must not configure a port:
```
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  base: './',
  plugins: [vue()],
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url))
    }
  }
})
```

2) Use hash mode for routing to avoid server-side routing configuration issues:
```
import { createRouter, createWebHashHistory } from 'vue-router'

const router = createRouter({
  history: createWebHashHistory(),
  routes: [
    // route definitions
  ]
})
```

3) Reference package.json:
```
{
  "scripts": {
    "dev": "vite",
    "build": "vite build"
  },
  "dependencies": {
    "vue": "^3.3.4",
    "vue-router": "^4.2.4"
  },
  "devDependencies": {
    "@vitejs/plugin-vue": "^4.2.3",
    "vite": "^4.4.5"
  }
}
```
